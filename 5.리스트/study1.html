리스트

홈페이지에서 구조와 형태가 비슷한 연관되어있는 컨텐츠를 표기하는 방법

li태그는 단독적으로 사용할 수 없다. 반드시 순서의 흐름을 고려하여 순서가 있다면 부모태그에 ol태그를, 없다면 부모태그에 ul태그를 사용해야 한다.

ol/ul태그의 자식태그로는 li태그만 사용할 수 있다.

li태그 안에는 또다른 문맥들이 형성될 수 있다. 그동안 배운 제목, 설명태그를 포함해 여러가지 태그 문맥이 li태그 안에서 사용될 수 있다.

Ordered list - ol

연관된 컨텐츠 즉 리스트로 표현되는 컨텐츠 사이에 순서의 영향이 있는 경우 ol태그를 부모태그로 사용한다.(숫자)

git 저장소 업데이트 순서

  1. git add .
  2. git commit -m "message"
  3. git push origin master

라면 끓이는 순서

  1. 물을 끓인다
  2. 스프를 넣는다
  3. 면을 넣는다
  4. 계란을 넣는다

Unordered list - ul

연관되어 있는 컨텐츠 즉 li태그들이 서로 순서의 영향이 없을 때 ul태그를 부모태그로 사용한다.
